# ONBOARDING（新手起步指南）

欢迎来到 **vibe-coding-lab**！

这份文档给第一次接触这个仓库的同学准备：
- 这个仓库是做什么的
- 怎么在本地跑起来
- 怎么在 Codex 里提任务
- 怎么验证你改的东西是生效的

---

## 1) 这个仓库是什么？

这是一个用于练习“Vibe Coding”协作流程的小实验仓库。
目前包含一个最小前端示例页面：
- 页面显示 `Hello Vibe Coding`
- 有一个“点击计数”按钮
- 计数会保存到浏览器 `localStorage`，刷新页面不会丢

它的目标不是复杂功能，而是让你快速上手：
1. 看懂仓库
2. 能运行
3. 能提需求
4. 能验证改动

---

## 2) 怎么运行？（本地 1 分钟）

这个示例不依赖任何框架，直接用浏览器就能打开。

### 方式 A：直接打开文件（最简单）
1. 进入仓库目录。
2. 双击 `index.html`，或在浏览器中打开它。

### 方式 B：用本地静态服务（推荐）
如果你本机有 Python：

```bash
python3 -m http.server 8000
```

然后访问：

```text
http://localhost:8000/index.html
```

> 为什么推荐方式 B？
> - 更接近真实开发访问方式
> - 后续加资源文件时更稳定

## 3) 怎么在 Codex 里提任务？

你可以把任务拆成“目标 + 验收标准 + 限制条件”。

建议模板（可直接复制）：

```text
请在这个仓库完成以下任务：
1) 目标：...
2) 验收标准：
   - ...
   - ...
3) 限制条件：
   - 不要引入新框架
   - 保持文件结构简单
4) 输出要求：
   - 给出修改说明
   - 给出验证命令
   - 提交 commit 并附 PR 信息
```

### 示例任务

```text
请在 index.html 增加一个“重置计数”按钮：
- 点击后将计数置为 0
- localStorage 同步更新
- 保持页面风格一致
- 最后告诉我怎么验证
```

## 4) 怎么验证改动？

每次改动后建议做这 4 步：

1. **页面打开正常**：浏览器能打开页面，没有白屏。
2. **功能可操作**：点击按钮后计数变化符合预期。
3. **刷新后不丢数据**：刷新页面后计数仍保留。
4. **控制台无报错**：打开开发者工具确认无红色错误。

### 针对当前示例的验证清单

- 打开 `index.html`
- 页面看到标题：`Hello Vibe Coding`
- 点击“点我 +1”后数字增长
- 刷新页面后数字保持不变
- 关闭页面再打开，数字仍保留（同一浏览器）

## 5) 常见问题（FAQ）

### Q1：我刷新后计数丢了？
可能原因：
- 浏览器开了无痕/隐私模式
- 手动清除了站点数据
- 改了 key 名称导致读不到旧值

### Q2：为什么不用 React/Vue？
这是“最小可运行示例”，目的是让新手先理解流程，再逐步引入框架。

### Q3：我想继续扩展，先做什么？
建议顺序：
1. 增加“重置按钮”
2. 增加“减少按钮”
3. 增加“步长输入框”
4. 再考虑引入构建工具
